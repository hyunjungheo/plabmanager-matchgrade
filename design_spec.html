진행 유력 매치 표시 시스템

매니저가 매치 리스트에서 "진행 가능성이 높은 매치"를 한눈에 알아보고 선택할 수 있도록, 매치 상태를 직관적인 마크로 표시하는 디자인 체계입니다.

최종 업데이트 2026.06 · 매니저 앱 매치 리스트 / 매치 상세

1개요

매치 리스트는 정보 밀도가 높고 카드 폭이 좁습니다(약 158px). 이 제약 안에서 매니저가 스캔만으로 매치의 진행 상태·거래 조건을 파악하도록, 학습이 필요 없는 작은 마크로 신호를 전달합니다.

  • 목표 · 진행률 높은 매치로 매니저 선택을 자연스럽게 유도해 매치 취소율을 낮춤
  • 원칙 · 색이 들어가면 "매치 상태", 무채색이면 "거래 정보"로 읽히게 분리
  • 제약 · 카드 박스 크기는 실제 앱과 동일하게 유지. 마크는 기존 요소를 밀어내지 않는 최소 크기

4상호배타 · 우선순위

  • 상태는 상호배타 · 진행 유력 / 신규 / 확정 / 긴급양도는 한 카드에 최대 1개
  • MY 오버라이드 · MY 매치는 상태·가격·확정 등 다른 모든 배지를 숨기고 MY만 노출
  • 거래 정보는 병기 가능 · 가격은 상태 마크와 함께 표시(예: 진행 유력 + 5,000원)
  • 주의 케이스 · 긴급양도 + 가격이 동시에 뜨면 시간 줄이 빠듯 → 실제 적용 시 "긴급양도 표시되면 가격 숨김" 우선순위 규칙 권장

6컴포넌트

① 매치 상태 표시 안내 모달

리스트 필터 아래 "매치 상태 표시 안내 ⓘ"로 진입. 진행 유력 · 신규 · 확정 마크의 의미를 설명합니다. 신규 라벨을 처음 본 사용자에게 1회 자동 노출되며(이후 미노출), 바텀시트 형태로 슬라이드업됩니다.

② 정렬 옵션

"정렬 ▾" 칩 → 바텀시트로 4개 옵션 제공.

기본 시작 임박 · 긴급양도/프로모션 확정 매치 배지 있는 매치

③ 매치 상세 페이지 — 배지 대신 섹션

상세 페이지에서는 진행 유력·신규 배지를 노출하지 않고, 대신 설명 섹션으로 안내합니다. 정보를 읽는 맥락(상세)에서는 충분한 설명이 더 적합하기 때문입니다.

  • 진행 유력 매치 섹션 · 주황 화살표 + "최근 5주 동안 진행이 많이 된 매치예요."
  • 신규 매치 섹션 · 노란 N + "최근에 새로 열린 매치예요. 아직 신청 인원이 많지 않을 수 있어요."

7주요 의사결정 기록

왜 점·아이콘이 아니라 기존 주황 화살표인가?

앱에 이미 쓰이던 표시라 매니저에게 학습 비용이 0. 새 기호를 발명해 가르치는 것보다 의미가 박힌 표시를 재사용하는 게 인지 부담이 가장 낮습니다.

강/약을 색 농도로 주지 않은 이유?

흐린 주황은 흰 배경에서 묻혀 가시성이 떨어집니다. 같은 진한 주황을 유지하고 꼬리 라인 수(2/1)로 강도를 표현해, "기세가 셀수록 선이 많다"는 직관과 가시성을 동시에 확보.

신규를 노란 N으로 한 이유?

"NEW=노랑/골드"는 가장 널리 통하는 직관. 한 글자 "N"으로 폭을 최소화하고, 팔레트 내 옐로(#FFE875)+앰버(#97500F)로 가독성 확보. 주황 화살표와는 색·형태가 모두 달라 혼동 없음.

MY가 다른 배지를 모두 가리는 이유?

내 매치는 이미 아는 정보. 거기에 다른 상태까지 겹치면 시각적으로 산만. MY 단독 노출로 클러터를 제거.

색이 많아지는 문제(레인보우)는?

색은 상태 신호(진행유력·신규·확정·긴급양도)에만 쓰고, 거래 정보(가격·MY)는 무채색으로 분리. "색=상태, 무채색=거래"로 색의 역할을 명확히 해 산만함을 통제.
